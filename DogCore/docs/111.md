## 2026-06-29 — 肢体控制层升级：方向映射 + 关节限制 + 踢腿测试

### 一、`app_limb.c` — 核心变更

#### 1.1 配置结构体新增字段（`limb_cfg_entry_t`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `dir_sign[2]` | `int8_t` | 方向符号，`+1`=IK 正向→舵机顺时针，`-1`=反向 |
| `joint_limit_min[2]` | `int16_t` | 关节角度下限 (°)，最终输出到舵机的钳位下限 |
| `joint_limit_max[2]` | `int16_t` | 关节角度上限 (°)，最终输出到舵机的钳位上限 |

#### 1.2 配置表 `LIMB_CFG[4]` 更新

```
腿 0: servo={1,2}, offset={0°, -90°}, dir_sign={+1,+1}, limit=[-90,90]
腿 1: servo={3,4}, offset={0°,   0°}, dir_sign={+1,+1}, limit=[-90,90]
腿 2: servo={5,6}, offset={0°,   0°}, dir_sign={+1,+1}, limit=[-90,90]
腿 3: servo={7,8}, offset={0°,   0°}, dir_sign={+1,+1}, limit=[-90,90]
```

> **腿 0 膝关节说明**：`offset=-90°` 补偿小腿平行于身体平面的安装偏角，`init_angle=90°` 使上电时小腿与大腿共线。

#### 1.3 函数修改

| 函数 | 修改内容 |
|------|----------|
| `Limb_SetTarget` | 目标角度钳位到 `joint_limit`；速度同步 delta 改用钳位后的值 |
| `Limb_SetFootTarget` | IK 角度公式改为 `ik_deg × dir_sign + offset` |
| `Limb_SetFootImmediate` | 同上 |
| `Limb_Scheduler` | 插值输出增加 `joint_limit` 钳位 |

**核心公式**：
```
servo_angle = IK解算角度(°) × dir_sign + offset_deg
              ↓ 钳位到 [joint_limit_min, joint_limit_max]
              → APP_Servo_SetTarget()
```

### 三、`app_gait_seq.h` — 踢腿序列 GAIT_IK_KICK

基于用户给定的舵机角度做**正向运动学**计算：

#### 参考舵机角度 → 足端坐标换算

| 姿态 | 舵机角度 | IK 角度（去 offset） | 足端坐标 (x, z) |
|------|----------|---------------------|-----------------|
| 最高点 | hip=-77°, knee=-63° | hip=-77°, knee=27° | **(-57.5, 27.4)** |
| 最低点 | hip=  0°, knee= 90° | hip=  0°, knee=180° | **(0, 260)** |

#### FK 推导过程

```
已知: L1=140mm, L2=120mm

最高点 (hip=-77°, knee=27°):
  d² = L1² + L2² - 2·L1·L2·cos(knee)
     = 140² + 120² - 2·140·120·cos(27°)
     = 34000 - 29937.8 = 4062.2
  d  = 63.74 mm

  α = atan2(L2·sin(knee), L1 + L2·cos(knee))
    = atan2(54.48, 246.92) = 12.40°

  φ = hip + α = -77° + 12.40° = -64.60°

  x = d·sin(φ) = 63.74 × (-0.9029) = -57.5 mm
  z = d·cos(φ) = 63.74 × 0.4299  =  27.4 mm

验证: IK_TwoLink(140, 120, -57.5, 27.4) → hip≈-77°, knee≈27° ✓
```

#### 踢腿序列（2 相位，循环）

| 相位 | 腿 0 足端 (x, z) | 腿 1~3 | 时长 |
|------|------------------|--------|------|
| 0 — 抬腿 | `(-57.5, 27.4)` | 保持 (0, 85) | 500ms |
| 1 — 落回 | `( 0.0, 260)` | 保持 (0, 85) | 500ms |

### 架构总览

```
UART 0xAA 命令
  └→ Gait_IK_Start(&GAIT_IK_KICK)
       └→ Gait_IK_ApplyPhase(phase)
            └→ Limb_SetFootTarget(leg_id, x, z, duration_ms)
                 ├→ IK_TwoLink(L1, L2, x, z) → hip_rad, knee_rad
                 ├→ hip_deg × dir_sign[0] + offset[0]  ─┐
                 ├→ knee_deg × dir_sign[1] + offset[1] ─┤ 钳位 joint_limit
                 └→ Limb_SetTarget(leg, joint_angles, duration)
                      ├→ 快照 start_joint = current_joint
                      ├→ target_joint = clamp(target, limit_min, limit_max)
                      ├→ 配速: duration = max(请求, 最慢关节全速时间)
                      └→ moving = 1
                           │
                           ▼ (每 10ms)
                      Limb_Scheduler()
                        ├→ t = elapsed / duration
                        ├→ angle = start + (target - start) × t
                        ├→ clamp(angle, limit_min, limit_max)
                        └→ APP_Servo_SetTarget(servo_id, angle)
                             │
                             ▼ (每 10ms)
                        APP_Servo_Scheduler()
                          └→ 浮点插值 → PWM 脉冲输出
```

### 六、坐标系字段重命名：`.z` → `.y`

**原因**：用户坐标系定义为 (x, y)，代码中混用 z 造成混淆，统一改为 y。

| 文件 | 修改内容 |

| `app_gait.h` | `ik_foot_pose_t` 成员 `z` → `y` |
| `app_limb.h` | `Limb_SetFootTarget` / `Limb_SetFootImmediate` 参数 `z` → `y` |
| `app_limb.c` | 同上，函数定义参数名同步修改 |
| `app_gait.c` | `Gait_IK_ApplyPhase` 中 `phase->legs[leg].z` → `.y` |
| `app_gait_seq.h` | 所有步态数据 `.z =` → `.y =`，注释同步更新 |

### 七、踢腿序列修正：x=0 纯垂直

**问题**：之前 x=-57.5，来自 FK(hip=-77°, knee=27°)。髋关节转了 77°，足端必然后移 57.5mm。

**分析**：用户的舵机角度 (-77°, -63°) 是直接舵机命令（F1），在物理机器人上观察到足端没有前后偏移。这说明 IK 模型的 x 轴与物理髋关节旋转平面不完全对齐。对于纯垂直踢腿，应令 x=0。

**修正**：

| 项目 | 修正前 | 修正后 |
|------|--------|--------|
| 最高点坐标 | `(-57.5, 27.4)` | `(0, 30)` |
| 最低点坐标 | `(0, 260)` | `(0, 260)` 不变 |
| IK 反解最高点 | hip=-77°, knee=27° | hip≈-5°, knee≈10° |
| 对应舵机角 | hip=-77°, knee=-63° | hip≈-5°, knee≈-80° |

**正向运动学验证**（x=0, y=30）：
```
IK_TwoLink(140, 120, 0, 30):
  d = 30
  cos_knee = (140²+120²-30²)/(2·140·120) = 33100/33600 = 0.9851
  knee = acos(0.9851) = 9.9°
  hip = atan2(0,30) - atan2(120·sin(9.9°), 140+120·cos(9.9°))
      = 0 - atan2(20.6, 258.2) = -4.6°

servo_hip = (-4.6°) × 1 + 0   = -5°
servo_knee = (9.9°) × 1 + (-90) = -80°  （均在 joint_limit [-90,90] 内 ✓）
```

> **注意**：修正后的舵机角 (-5°, -80°) 和用户 F1 测试的 (-77°, -63°) 差别较大。
> 这是因为 F1 的髋关节转角 (-77°) 包含了大幅后摆分量。
> 若物理上 x=0 的踢腿效果不理想，可通过调 x 非零值让髋关节参与动作。
