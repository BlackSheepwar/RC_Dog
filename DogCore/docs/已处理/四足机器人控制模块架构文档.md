# 四足机器人控制模块架构文档

> 基于二连杆逆运动学的步态控制系统，采用**分层架构**（步态层 → 肢体层 → IK 数学层 → 舵机层）。

---

## 一、架构总览

```
┌───────────────────────────────────────────────────────────────┐
│                    步态控制层 (app_gait)                       │
│  定义足端笛卡尔轨迹步态、相位推进、调度 IK 解算               │
│  数据：ik_gait_phase_t (足端 x/z + 持续时间)                  │
├───────────────────────────────────────────────────────────────┤
│                    肢体控制层 (app_limb)                       │
│  时间基线性插值，每个肢体独立状态机                           │
│  数据：关节角度 (int16_t[LIMB_JOINT_COUNT])                   │
├───────────────────────────────────────────────────────────────┤
│                  逆运动学层 (ik_two_link)                      │
│  纯数学解算：足端坐标 (x,z) → 髋/膝关节角度 (rad)            │
│  无状态，纯函数                                               │
├───────────────────────────────────────────────────────────────┤
│                    舵机控制层 (app_servo)                      │
│  浮点角度平滑 + 角速度驱动，最终输出 PWM 脉宽                │
│  数据：舵机角度 → 脉宽 (μs) → BSP_PWM_SetPulseUs             │
└───────────────────────────────────────────────────────────────┘
```

---

## 二、数据流

```
用户选择步态 (GAIT_IK_STAND / GAIT_IK_SIT / GAIT_IK_WALK)
       │
       ▼
Gait_IK_Start(gait_sequence)
       │
       ▼  (每 tick: Gait_IK_Scheduler)
判断当前相位时间到? ──否──→ Limb_Scheduler 继续插值
       │是                       │
       ▼                        ▼
IK_TwoLink(x, z → hip, knee)  线性插值 angle = start + Δ × t
       │                        │
       ▼                        ▼
Limb_SetTarget(角度, 时间) ──→ APP_Servo_SetTarget(角度)
                                      │
                                      ▼
                              APP_Servo_Scheduler
                              (浮点平滑步进 + 脉宽转换)
                                      │
                                      ▼
                              BSP_PWM_SetPulseUs(脉宽)
```

## 三、各模块详解

### 3.1 逆运动学层 — `ik_two_link`（纯数学）

**文件：** [ik_two_link.c](../User/Middlewares/IK/ik_two_link.c) / [ik_two_link.h](../User/Middlewares/IK/ik_two_link.h)

**职责：** 给定连杆长度和足端坐标，解算髋/膝关节角度。

#### 核心 API

| 函数 | 说明 |
|------|------|
| `IK_TwoLink(L1, L2, x, z, &hip, &knee)` | 二连杆逆运动学解算。返回 1=可达，0=不可达 |

#### 解算算法

```
输入：L1(大腿长), L2(小腿长), x(前后), z(竖直)
1. d = sqrt(x² + z²)，检查是否在 [|L1-L2|, L1+L2] 内
2. 余弦定理求膝角: cos_knee = (L1² + L2² - d²) / (2·L1·L2)
   knee = acos(cos_knee)
3. 几何法求髋角: hip = atan2(x, z) - atan2(L2·sin(knee), L1 + L2·cos(knee))
```

**关键常量：** `RAD_TO_DEG` (弧度→度), `DEG_TO_RAD` (度→弧度)

**注意：** 本模块**无内部状态**，每次调用独立解算。返回值 0 表示目标超出工作空间，调用方需处理（跳过/保持原位）。

### 3.2 肢体控制层 — `app_limb`（时间基插值）

**文件：** [app_limb.c](../User/APP/APP_Limb/app_limb.c) / [app_limb.h](../User/APP/APP_Limb/app_limb.h)

**职责：** 管理肢体状态机，对每个关节做**时间基线性插值**，将插值结果写入舵机层。

#### 数据结构

```c
typedef struct {
    uint8_t servo_id[LIMB_JOINT_COUNT];              // 关节→舵机 ID 映射
    int16_t current_joint[LIMB_JOINT_COUNT];         // 当前插值角度
    int16_t start_joint[LIMB_JOINT_COUNT];           // 本次运动起始角度
    int16_t target_joint[LIMB_JOINT_COUNT];          // 目标角度
    uint32_t start_tick;                              // 运动起始 tick (ms)
    uint32_t duration_ms;                             // 运动持续时间 (ms)
    uint8_t registered : 1;                           // 1=已注册
    uint8_t moving : 1;                               // 1=正在插值
} limb_t;
```

#### API 清单

| 函数 | 说明 | 被谁调用 |
|------|------|----------|
| `Limb_Init()` | 初始化（清零全部状态） | `Gait_IK_Init()` |
| `Limb_Register(id, servo_id[])` | 注册肢体 → 舵机映射 | `Gait_IK_Init()` |
| `Limb_SetTarget(id, target[], duration_ms)` | 设置目标角度 + 过渡时间，自动快照起始角度 | `Gait_IK_ApplyPhase()` |
| `Limb_SetImmediate(id, target[])` | 强制跳转到指定角度（无插值） | 外部命令 |
| `Limb_IsMoving(id)` | 查询单肢体运动状态 | 外部查询 |
| `Limb_IsAnyMoving()` | 查询是否有任何肢体在运动 | 外部查询 |
| `Limb_GetCurrent(id, out[])` | 获取当前插值角度 | 外部查询 |
| `Limb_Stop(id)` | 停止单肢体运动（停留在当前角度） | — |
| `Limb_StopAll()` | 停止所有肢体运动 | `Gait_IK_Stop()` |
| `Limb_Scheduler()` | **插值调度器**（每 tick 调用） | `task_servo.c` |

#### 核心逻辑（Limb_Scheduler）

```c
// 每 SERVO_TICK_MS 执行一次（通常 10ms）
for each 注册的肢体:
    if not moving → skip
    
    elapsed = now - start_tick
    t = min(elapsed / duration_ms, 1.0)   // 归一化进度 [0,1]
    
    for each 关节 j:
        angle = start_joint[j] + (target[j] - start_joint[j]) * t
        APP_Servo_SetTarget(servo_id[j], angle)
    
    if t >= 1.0 → moving = 0
```

**设计要点：**
- 使用 `HAL_GetTick()` 时间基，自动处理 49 天溢出（unsigned wrap-around）
- 每个肢体独立状态机，互不干扰
- 若 `target == current`，立即标记 `moving=0`，跳过插值

### 3.3 步态控制层 — `app_gait`

**文件：** [app_gait.c](../User/APP/APP_Gait/app_gait.c) / [app_gait.h](../User/APP/APP_Gait/app_gait.h)

**职责：** 定义步态序列（足端笛卡尔轨迹），按时间推进相位，调用 IK 解算后写入 Limb 层。

#### 数据结构

```c
// 足端坐标（相对髋关节）
typedef struct {
    float x;   // 前后（+前）
    float z;   // 竖直（+下）
} ik_foot_pose_t;

// 一个步态相位（所有腿的目标 + 持续时长）
typedef struct {
    ik_foot_pose_t legs[LEG_COUNT];  // 每条腿的足端位置
    uint32_t       duration_ms;       // 过渡时间
} ik_gait_phase_t;

// 步态序列
typedef struct {
    const ik_gait_phase_t *phases;   // 相位数组
    uint8_t                phase_count;
    uint8_t                loop;      // 0=单次, 1=循环
} ik_gait_sequence_t;
```

#### API 清单

| 函数 | 说明 | 内部调用链 |
|------|------|-----------|
| `Gait_IK_Init()` | 初始化：Limb_Init() + 注册肢体 + 设舵机高速 | → `Limb_Init()` → `Limb_Register()` → `APP_Servo_SetSpeed()` |
| `Gait_IK_Start(gait)` | 开始执行步态，立即解算第一相位 | → `Gait_IK_ApplyPhase()` |
| `Gait_IK_Stop()` | 停止步态 | → `Limb_StopAll()` |
| `Gait_IK_IsRunning()` | 查询运行状态 | — |
| `Gait_IK_Scheduler()` | **步态调度器**（每 tick 调用） | → `Gait_IK_ApplyPhase()`（到时才调用） |

#### 核心逻辑（Gait_IK_Scheduler）

```c
if not active → return

elapsed = now - phase_start_tick
if elapsed < phase_duration → return  // 未到时间，Limb_Scheduler 继续插值

// 推进相位
phase_index++
if phase_index >= phase_count:
    if loop → phase_index = 0
    else    → active = 0, return

// 解算新相位
Gait_IK_ApplyPhase(&phases[phase_index])
```

#### 内部关键函数

**`Gait_IK_ApplyPhase(phase)`** — 对每条腿执行完整链路：

```c
for each leg:
    if IK_TwoLink(L1, L2, x, z, &hip, &knee):
        target[0] = hip_rad  × RAD_TO_DEG  → 髋关节角度
        target[1] = knee_rad × RAD_TO_DEG  → 膝关节角度
        Limb_SetTarget(leg, target, duration_ms)
    else:
        continue  // 不可达 → 保持原位
```

#### 内置步态序列

| 步态 | 类型 | 相位 | 说明 |
|------|------|------|------|
| `GAIT_IK_STAND` | 循环 | 1 个 | 双腿直立，z=85mm，膝关节微屈 |
| `GAIT_IK_SIT` | 单次 | 1 个 | 双腿弯曲到 z=45mm，500ms 过渡 |
| `GAIT_IK_WALK` | 循环 | 4 个 | Trot 步态：抬膝→前摆→换腿→前摆 |

**Walk 步态相位详解：**

| 相位 | 左腿 | 右腿 | 时长 |
|------|------|------|------|
| 0 | 抬膝 (z=55) | 支撑 (x=15, z=80) | 150ms |
| 1 | 前摆 (x=30, z=55) | 支撑后移 (x=0, z=80) | 200ms |
| 2 | 落地支撑 (x=0, z=80) | 抬膝 (z=55) | 150ms |
| 3 | 支撑后移 (x=-10, z=80) | 前摆 (x=30, z=55) | 200ms |

#### 几何参数

```c
#define LEG_L1  140.0f     // 大腿长度（髋→膝），单位 mm
#define LEG_L2  180.0f     // 小腿长度（膝→足），单位 mm
#define LIMB_SERVO_SPEED_DPS  5000.0f  // 舵机跟随速度 (°/s)
```

### 3.4 舵机控制层 — `app_servo`

**文件：** [app_servo.c](../User/APP/APP_Servo/app_servo.c) / [app_servo.h](../User/APP/APP_Servo/app_servo.h)

**职责：** 单个舵机的速度驱动平滑控制，最终输出 PWM。

#### API 清单

| 函数 | 说明 | 被谁调用 |
|------|------|----------|
| `APP_Servo_Init()` | 初始化舵机池 + BSP PWM | 系统启动 |
| `APP_Servo_Add(id, hw, speed)` | 注册舵机实例 | 系统启动 |
| `APP_Servo_SetTarget(id, angle)` | 设置目标角度（限幅保护） | `Limb_Scheduler()`, `Limb_SetImmediate()` |
| `APP_Servo_SetincreaseTarget(id, delta)` | 增量设置目标角度 | CAN/USART 命令 |
| `APP_Servo_GetCurrent(id)` | 获取当前角度（取整） | 外部查询 |
| `APP_Servo_SetSpeed(id, dps)` | 设置角速度 | `Gait_IK_Init()` |
| `APP_Servo_Scheduler()` | **平滑调度器**（每 10ms 调用） | `task_servo.c` |

#### 核心逻辑（APP_Servo_Scheduler）

```c
for each 舵机:
    diff = target_angle - current_angle
    if diff 很小 → 直接到位，输出脉宽
    
    move = speed_dps × SERVO_TICK_S  // 每 tick 走多少度
    if move > |diff| → move = |diff|
    current_angle += (diff > 0) ? move : -move
    
    // 角度 → 脉宽输出
    pulse = pulse_min + (current + offset + half_range) × (pulse_max - pulse_min) / phys_range
    BSP_PWM_SetPulseUs(id, pulse)
```

## 五、调度时序

所有调度器均被 `task_servo.c` 的任务函数按固定周期调用：

```
每 10ms (SERVO_TICK_MS):
  1. Gait_IK_Scheduler()    → 检查相位推进
  2. Limb_Scheduler()       → 检查所有肢体插值
  3. APP_Servo_Scheduler()  → 输出 PWM 到硬件
```

**注意三层调度器的职责分离：**
- `Gait_IK_Scheduler`：只管"当前相位到没到时间"，到了就解算下一相位并设 Limb 目标
- `Limb_Scheduler`：只管"当前肢体做线性插值"，不停写 `APP_Servo_SetTarget`
- `APP_Servo_Scheduler`：只管"按 speed_dps 浮点平滑到目标角度"，输出 PWM

三层可以独立运行，互不阻塞。
