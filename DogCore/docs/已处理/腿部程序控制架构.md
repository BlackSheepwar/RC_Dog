# 肢体控制系统重构方案

## 现状分析

当前代码分为两层：

```
┌─────────────────────────────────────┐
│  Gait 层 (旧: app_servo_gait.c)          │
│  - 相位表：每个相位=所有腿的目标角度  │
│  - 硬编码角度值，无运动学计算        │
│  - 按 tick 计数推进相位             │
├─────────────────────────────────────┤
│  Servo 层 (app_servo.c)            │
│  - 单舵机角度平滑（速度驱动）        │
│  - 每 tick 移动 speed_dps * 0.01   │
└─────────────────────────────────────┘
```

**问题：**
1. Gait 层直接设舵机目标，跳过了时间维度的平滑调度
2. 相位持续用 tick 数表示，不易直观控制速度/时间
3. 关节角度硬编码，改动步态要重新标定每个关节
4. 无运动学逆解，无法在笛卡尔空间规划足端轨迹

---

## 目标架构

```
┌──────────────────────────────────────────────┐
│  Gait / Motion 层 (新)                       │
│  - 足端轨迹生成（笛卡尔空间）                 │
│  - 两连杆运动学逆解 (IK)                      │
│  - 步态相位描述足端位置，而非关节角度          │
├──────────────────────────────────────────────┤
│  Limb 控制层 (新)                            │
│  - 维护肢体状态（当前/目标角度 + 到达时间）    │
│  - 时间基插值调度（非 tick 累计）             │
│  - 每个肢体的独立运动控制                     │
├──────────────────────────────────────────────┤
│  Servo 层 (app_servo.c，不变)               │
│  - 单舵机角度 → PWM 脉宽输出                 │
└──────────────────────────────────────────────┘
```

---

## 方案一（推荐）：二连杆 IK + 时间调度 Limb 控制

### Limb 控制层（取代 gait 的直接相位推进）

**数据结构：**

```c
// app_limb.h

#define SERVO_PER_LEG    2    /* 髋、膝 */

/** @brief 肢体状态：时间基插值运动 */
typedef struct {
    /* —— 当前插值状态 —— */
    int16_t current_joint[SERVO_PER_LEG];   /* 当前插值角度 */
    
    /* —— 本次运动目标 —— */
    int16_t start_joint[SERVO_PER_LEG];     /* 起始角度（设目标时快照） */
    int16_t target_joint[SERVO_PER_LEG];    /* 目标角度 */
    uint32_t start_tick;                    /* 运动起始时刻（ms） */
    uint32_t duration_ms;                   /* 运动持续时间（ms） */
    uint8_t  moving;                        /* 1=运动中, 0=已到位 */

    /* —— 舵机 ID 映射 —— */
    uint8_t servo_id[SERVO_PER_LEG];
} limb_t;
```

**调度逻辑（每 10ms）：**

```c
void Limb_Scheduler(void)
{
    uint32_t now = osKernelGetTickCount();  /* 或 HAL_GetTick */

    for (uint8_t i = 0; i < limb_count; i++)
    {
        limb_t *limb = &limbs[i];
        if (!limb->moving) continue;

        /* 1. 计算归一化进度 t ∈ [0, 1] */
        uint32_t elapsed = now - limb->start_tick;
        float t = (float)elapsed / (float)limb->duration_ms;
        if (t >= 1.0f) t = 1.0f;

        /* 2. 线性插值每个关节 */
        for (uint8_t j = 0; j < SERVO_PER_LEG; j++)
        {
            float range = (float)(limb->target_joint[j] - limb->start_joint[j]);
            limb->current_joint[j] = (int16_t)(limb->start_joint[j] + range * t + 0.5f);
            APP_Servo_SetTarget(limb->servo_id[j], limb->current_joint[j]);
        }

        /* 3. 到位标记 */
        if (t >= 1.0f) limb->moving = 0;
    }
}
```

**API：**

```c
/** @brief 初始化肢体控制 */
void Limb_Init(void);

/**
 * @brief 注册一条肢体（指定舵机映射）
 * @param limb_id    肢体编号
 * @param servo_id   舵机ID数组（长度 SERVO_PER_LEG）
 */
void Limb_Register(uint8_t limb_id, const uint8_t servo_id[SERVO_PER_LEG]);

/**
 * @brief 设置肢体目标角度（时间基插值）
 * @param limb_id     肢体编号
 * @param target      目标关节角度数组
 * @param duration_ms 过渡时间（ms）
 * @note 自动快照当前角度为起始值，开始时间基插值
 */
void Limb_SetTarget(uint8_t limb_id, const int16_t target[SERVO_PER_LEG], uint32_t duration_ms);

/**
 * @brief 强制跳转到指定角度（瞬间到位）
 */
void Limb_SetImmediate(uint8_t limb_id, const int16_t target[SERVO_PER_LEG]);

/**
 * @brief 查询肢体是否运动完毕
 * @retval 1 运动中
 */
uint8_t Limb_IsMoving(uint8_t limb_id);

/**
 * @brief 获取当前插值角度
 */
void Limb_GetCurrent(uint8_t limb_id, int16_t out[SERVO_PER_LEG]);

/**
 * @brief 停止肢体运动（保持在当前位置）
 */
void Limb_Stop(uint8_t limb_id);
```

### 上层：两连杆 IK 步态发生器

**运动学模型：**

```
       髋关节 (hip)
         ●
        /|\
       / | \ L1 (大腿)
      /  |  \
     ●───●───●  膝关节 (knee)
      \  |  /
       \ | / L2 (小腿)
        \|/
         ●
      足端 (foot)
      
      坐标系：髋关节为原点
      x: 前后方向（+前）
      z: 竖直方向（+下）
```

**IK 解算：**

```c
/** @brief 二连杆逆运动学
 *  @param L1    大腿长度
 *  @param L2    小腿长度
 *  @param x     足端 x 坐标（前后）
 *  @param z     足端 z 坐标（竖直，向下为正）
 *  @param hip   [out] 髋关节角度
 *  @param knee  [out] 膝关节角度
 *  @retval 1 解有效（可达）
 *  @retval 0 不可达
 */
uint8_t IK_TwoLink(float L1, float L2, float x, float z,
                   float *hip, float *knee)
{
    float d_sq = x * x + z * z;
    float d = sqrtf(d_sq);

    /* 不可达检查 */
    if (d > L1 + L2 || d < fabsf(L1 - L2)) return 0;

    /* 膝角（余弦定理） */
    float cos_knee = (L1*L1 + L2*L2 - d_sq) / (2.0f * L1 * L2);
    *knee = acosf(cos_knee);

    /* 髋角 */
    float alpha = atan2f(x, z);
    float beta = atan2f(L2 * sinf(*knee), L1 + L2 * cosf(*knee));
    *hip = alpha - beta;

    return 1;
}
```

**足端轨迹 + 步态生成：**

```c
// IK 步态相位（描述足端位置，非关节角度）
typedef struct {
    float foot_x;           // 足端 x（前后），相对髋关节
    float foot_z;           // 足端 z（竖直），相对髋关节  
} ik_foot_pose_t;

typedef struct {
    ik_foot_pose_t legs[LEG_COUNT];   // 所有腿的足端位置
    uint32_t       duration_ms;       // 过渡时间（ms）
} ik_gait_phase_t;

// 步态：一系列 IK 相位
typedef struct {
    const ik_gait_phase_t *phases;
    uint8_t                phase_count;
    uint8_t                loop;
} ik_gait_sequence_t;
```

**步态调度器：**

```c
void Gait_IK_Scheduler(void)
{
    if (!s_ik_runner.active) return;

    /* 当前相位 */
    const ik_gait_phase_t *phase = /* ... */;

    for (uint8_t leg = 0; leg < LEG_COUNT; leg++)
    {
        float hip_angle, knee_angle;
        
        if (IK_TwoLink(L1, L2,
                       phase->legs[leg].foot_x,
                       phase->legs[leg].foot_z,
                       &hip_angle, &knee_angle))
        {
            int16_t target[2] = {
                (int16_t)(hip_angle * RAD_TO_DEG + 0.5f),
                (int16_t)(knee_angle * RAD_TO_DEG + 0.5f)
            };
            Limb_SetTarget(leg, target, phase->duration_ms);
        }
    }

    /* 推进到下一相位 */
    AdvancePhase();
}
```

**用足端轨迹描述"行走"（Trot）：**

```
相位 0: 左腿抬 (x=0,  z=-20)  右腿支撑 (x=30, z=0)
相位 1: 左腿前摆 (x=40, z=-20) 右腿支撑 (x=30, z=0)
相位 2: 右腿抬 (x=60, z=-20)  左腿支撑 (x=30, z=0)  
相位 3: 右腿前摆 (x=100,z=-20) 左腿支撑 (x=30, z=0)
```

IK 自动算出髋/膝关节角度，不再手动标定。

## 方案二：无极插值 + 柔顺控制（进阶）

在方案一基础上增加：

### 插值曲线配置

```c
typedef enum {
    INTERP_LINEAR,      // 线性（默认）
    INTERP_EASE_IN,     // 缓入: t²
    INTERP_EASE_OUT,    // 缓出: 1-(1-t)²
    INTERP_SMOOTHSTEP,  // 平滑: t²(3-2t)
    INTERP_ACCEL_DECEL, // 匀加/减速
} interp_profile_t;
```

缓出对落腿更自然（轻轻着地），缓入对抬腿更自然。

### 可叠加的周期性摆动

```c
typedef struct {
    uint8_t  enable;        // 是否叠加
    float    amplitude;     // 摆动幅度 (°)
    float    frequency;     // 摆动频率 (Hz)
    float    phase_offset;  // 相位偏移
} limb_swing_t;
```

用于模拟弹簧阻尼、小跑时自然振动等。

## 方案三：状态机步态编排器（高可扩展）

在 limb 控制 + IK 之上再加一层**有限状态机**编排：

```
     Idle
      │
      ▼
   ┌─────┐  trigger ┌────────┐
   │Stand├─────────►│SitDown │
   └─────┘          └────────┘
      │                │
      │trigger         │done
      ▼                ▼
   ┌─────┐          ┌────────┐
   │Walk ├─────────►│SitDone │
   └─────┘          └────────┘
      │
      │trigger
      ▼
   ┌─────┐
   │Trot │
   └─────┘
```

状态切换时，Limb 层的 time-based 插值保证过渡平滑，无需额外代码。

## 推荐路线

| 步骤 | 内容 | 交付 |
|------|------|------|
| **1** | 实现 Limb 时间调度层（方案一的下层） | `app_limb.c/h` |
| **2** | 移植现有步态数据到 Limb API（线性过渡） | 不改步态内容，只改调用方式 |
| **3** | 实现二连杆 IK（方案一的上层） | `ik_two_link.c/h` |
| **4** | 用 IK 表达足端轨迹步态 | 替代硬编码关节角度 |
| **5** | 可选：插值曲线增强 | 更自然运动 |
| **6** | 可选：状态机编排 | 多步态间平滑切换 |

## 核心收益

1. **时间维度的语义化**：传参用 `duration_ms`，不用 `phase_tick × 10ms`
2. **肢体独立控制**：每条腿可独立设目标，不再只能整机同步跳相位
3. **笛卡尔空间编程**：用 IK 描述足端位置，无需手动推导关节角度
4. **自然抗抖**：时间基插值不受 tick 偏移影响，到位即停
5. **可扩展**：插值曲线、摆幅叠加、状态机编排都是独立模块
